# Actor Object

The `Actor` object represents any entity in the game world that has a physical presence, such as characters, items, or environmental elements. This guide provides a detailed explanation of how to interact with `Actor` objects using Lua scripting in your game.

## Getting Started

To interact with an `Actor` in Lua, you first need to obtain a reference to it. This can be done through various game-specific mechanisms, such as collision events, overlap events, or by querying the game world.

**Example:**

```lua
-- Assume 'Actor' is a valid Actor object obtained from the game
```

Once you have an `Actor` reference, you can use the methods described below to manipulate or retrieve information about it.

---

## Methods

### GetLocation()

**Description:**

Returns the current location of the `Actor` in the game world.

**Syntax:**

```lua
local position = Actor:GetLocation()
```

**Returns:**

- `position`: A table containing the fields `x`, `y`, and `z`, representing the `Actor`'s position in world coordinates.

**Example:**

```lua
local position = Actor:GetLocation()
print("Actor Position:", position.x, position.y, position.z)
```

---

### SetLocation(position)

**Description:**

Sets the `Actor`'s location in the game world.

**Syntax:**

```lua
Actor:SetLocation(position)
```

**Parameters:**

- `position`: A table containing the fields `x`, `y`, and `z`, specifying the new position for the `Actor`.

**Example:**

```lua
local newPosition = { x = 100, y = 200, z = 300 }
Actor:SetLocation(newPosition)
```

## Example Usage

Here's an example that combines several of the above methods:

```lua
-- Obtain the Actor's current position
local position = Actor:GetLocation()

-- Move the Actor upward by 100 units
position.z = position.z + 100
Actor:SetLocation(position)

-- Apply an upward impulse to the Actor
local impulse = { x = 0, y = 0, z = 500 }
Actor:AddImpulse(impulse)

-- Check if physics is enabled
if not Actor:GetPhysicsEnabled() then
    -- Enable physics
    Actor:SetPhysicsEnabled(true)
end

-- Disable gravity
Actor:SetGravity(false)

-- Print the Actor's name and class
local name = Actor:GetName()
local className = Actor:GetClassName()
print("Actor:", name, "Class:", className)
```

## Notes

- The `Actor` object represents a single instance of an actor in the game world.
- Ensure that you have a valid `Actor` reference before calling these methods to avoid runtime errors.
- Changes to the `Actor`'s properties may have immediate effects on the game world and physics simulation.

## Troubleshooting

- **Method Not Found Error**: Ensure that you are calling methods on a valid `Actor` object.
- **Unexpected Behavior**: Verify that the parameters you pass to methods are in the correct format and contain all required fields.
